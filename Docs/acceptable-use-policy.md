# Acceptable Use Policy

## Purpose

This policy defines acceptable and prohibited behavior within the MB Tech Labs Work Management & Monitoring System. All users must comply with this policy.

---

## Acceptable Use

Users are expected to:

### Work-Related Use
- Use the platform exclusively for assigned work tasks
- Log time accurately and honestly
- Submit genuine work deliverables
- Communicate issues and delays transparently
- Follow task status and reporting procedures
- Collaborate professionally with team
- Complete assigned work to quality standards

### Professional Conduct
- Use respectful and professional language
- Follow company policies consistently
- Report concerns through proper channels
- Support team collaboration
- Maintain confidentiality
- Protect sensitive information
- Comply with all company rules

### Proper System Usage
- Use own credentials only (unique login)
- Keep password confidential
- Report security concerns
- Use system for its intended purpose
- Follow access control rules
- Respect data privacy
- Maintain system integrity

---

## Prohibited Behavior

The following actions are strictly prohibited and may result in disciplinary action or termination:

### Time Tracking Violations

**Fake or Inflated Time Logs**
- Creating false time entries
- Logging hours not actually worked
- Inflating hours beyond actual time
- Backdating entries more than permitted (7 days)
- Fabricating work sessions

**Timer Manipulation**
- Leaving timer running while not working
- Manipulating timer duration retrospectively
- Starting/stopping timer dishonestly
- Using timer for non-work purposes
- Sharing timer credentials with others

**Time Entry Fraud**
- Logging time for tasks not assigned
- Duplicate time entries for same work
- Logging time without manager approval (for manual entries)
- Back-dating entries improperly
- Deleting or modifying approved entries

**System Abuse**
- Keeping timer active while idle
- Passive timer usage to inflate hours
- Logging identical time patterns daily (suspicious regularity)
- Systematic under-reporting of actual work time
- Gaming the system to reach targets

### Task Management Violations

**Task Misuse**
- Creating false tasks or assignments
- Modifying task details inappropriately
- Submitting work not meeting acceptance criteria
- Marking incomplete tasks as done
- Claiming others' completed work

**Submission Fraud**
- Submitting fabricated deliverables
- Copying others' work without attribution
- Submitting incomplete work as complete
- Misrepresenting deliverable quality
- Plagiarizing code or content

### Communication & Client Policy Violations

**Unauthorized Client Communication**
- Interns/Employees contacting clients directly
- Unauthorized Team Leads communicating with clients
- Disclosing confidential project information
- Making commitments or promises on behalf of MB Tech Labs
- Discussing pricing or payment with clients
- Sharing internal processes or methodologies

**Confidentiality Breaches**
- Disclosing client information to unauthorized parties
- Sharing proprietary tools or methods
- Revealing other clients' information
- Discussing projects outside authorized team
- Posting company/client info on social media

**Scope Creep**
- Making unauthorized commitments
- Over-promising deliverables
- Changing requirements without approval
- Committing to timelines not approved

### Data & Security Violations

**Credential Misuse**
- Sharing login credentials with others
- Using another's account
- Sharing passwords
- Accessing systems without authorization
- Using API keys improperly

**Data Protection**
- Downloading client data for personal use
- Photographing or screenshotting confidential content
- Storing company data on personal devices
- Using unsecured/unfamiliar networks with company data
- Failing to protect company information

**System Access Abuse**
- Accessing other users' data without authorization
- Attempting to bypass security controls
- Modifying other users' task assignments
- Deleting or corrupting system data
- Accessing systems beyond permitted scope

### Conduct Violations

**Fraudulent Activity**
- Intentional deception or lying
- Misrepresenting identity or credentials
- False claims or fabricated information
- Dishonest practices

**Harassment & Discrimination**
- Abusive or disrespectful communication
- Discrimination based on protected characteristics
- Hostile work environment creation
- Bullying or intimidation
- Sexual or racial harassment

**Professional Misconduct**
- Insubordination or refusing directives
- Gross negligence or carelessness
- Working under influence of substances
- Creating hostile environment
- Violated NDA or confidentiality agreement

### System Misuse

**Unauthorized Access**
- Attempting to access restricted systems
- Bypassing access controls
- Escalating privileges improperly
- Accessing audit logs without authorization
- Circumventing security measures

**Data Corruption**
- Deleting or modifying system records
- Corrupting audit trails
- Manipulating reports or metrics
- Creating false records
- System sabotage

**Policy Circumvention**
- Intentionally violating policies
- Finding loopholes to avoid compliance
- Encouraging others to violate policies
- Retaliation for policy enforcement

## Enforcement & Investigation

### Monitoring
- System activity regularly reviewed
- Suspicious entries flagged automatically
- Pattern analysis conducted
- Admin and Managers monitor compliance
- Reports generated for review

### Investigation Process

**Detection**:
- Violation suspected or reported
- Evidence collected
- Audit trail reviewed
- Context gathered

**Investigation**:
- Facts determined
- Involved parties interviewed
- Additional evidence collected
- Witnesses spoken to
- Documentation reviewed

**Determination**:
- Violation confirmed or unconfirmed
- Severity assessed
- Context considered
- Decision made

**Action**:
- Appropriate consequence applied
- User notified
- Documentation recorded
- Appeals process available

---

## Consequences

### Minor Violations (First Offense)

**Examples**: Occasional manual time entry without approval, minor task status error

**Consequences**:
- Verbal warning from Team Lead
- Documented warning in file
- Required retraining or clarification
- 72-hour improvement period
- Continued monitoring

### Moderate Violations (Repeated or More Serious)

**Examples**: Repeated fake time entries, consistent task fraud, unauthorized client contact

**Consequences**:
- Written disciplinary notice
- Mandatory coaching or retraining
- Suspension of privileges (e.g., manual time entry)
- Stipend or refund withholding
- 1-2 week probation period
- Performance improvement plan

### Severe Violations (Major Misconduct)

**Examples**: Systematic time fraud, major client communication breach, confidentiality breach, falsified deliverables

**Consequences**:
- Immediate termination of engagement
- Disqualification from stipend/refund
- Potential legal action
- Report to authorities (if applicable)
- Escalation to HR/Admin for record
- Possible law enforcement involvement

### Criminal Activity

**Examples**: Data theft, hacking, intellectual property theft

**Consequences**:
- Immediate termination
- Law enforcement involvement
- Legal prosecution
- Civil litigation
- Restitution if applicable

---

## Special Circumstances

### Good Faith Mistakes
- Honest errors treated differently
- First-time, unintentional mistakes may result in warning only
- Pattern required for escalation
- Intent matters (accident vs. deliberate)

### System Errors
- Technical issues that cause false flags may be excluded
- Admin can adjust records if system error proven
- User reports system issues promptly
- Investigation conducted
- Correction applied if warranted

### Extenuating Circumstances
- Illness or personal emergency
- System outages or technical issues
- Legitimate reasons for exceptions
- Documented and approved
- Context considered
- Reasonable accommodations provided

---

## Appeals & Dispute Resolution

### Right to Appeal
- User can dispute violation finding
- Can provide additional context
- Request review of evidence
- Escalate through proper channels
- Due process guaranteed

### Appeal Process
1. Notify HR Manager of dispute
2. Request review within 5 business days
3. Present additional information
4. HR/Admin reviews case
5. Decision made and communicated
6. Final authority: Admin

### Retaliation Protection
- No retaliation for reporting violations
- No punishment for appealing decisions
- Protected from adverse action
- Retaliation is separate violation
- Report retaliation immediately

## Enforcement

### Monitoring
- Admin and Managers review time logs and activity regularly
- Suspicious entries are flagged and investigated
- Pattern analysis identifies repeated violations
- Reports are generated for compliance review

### Consequences
Violations may result in:

**Minor Violations** (first occurrence):
- Verbal or written warning
- Required retraining
- Suspension of feature access (e.g., manual time entry)
- 72-hour suspension pending review

**Moderate Violations** (repeated or more serious):
- Written disciplinary notice
- Mandatory coaching or performance improvement plan
- Stipend or refund withholding
- Temporary system access suspension
- 1-week probation period

**Severe Violations** (fraud, major misconduct):
- Immediate termination of engagement
- Disqualification from stipend/refund
- Legal action and recovery of damages
- Reporting to authorities if applicable

## User Responsibilities

- Use the start/stop timer correctly for assigned work
- Keep work sessions aligned to actual task progress
- Record accurate notes for manual or exception entries
- Report any system issues or concerns promptly
- Comply with all policies and directives
- Seek clarification before taking uncertain actions
