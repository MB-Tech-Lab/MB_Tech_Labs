# Work Monitoring & Tracking Policy

## Purpose

This policy describes how MB Tech Labs monitors work activity and time tracking to ensure accountability, transparency, and fair performance evaluation across the organization.

---

## Monitoring Principles

### What We Monitor
- **Task Assignment & Status**: What work is assigned and current progress
- **Time Tracking**: Hours logged and task-based time entries
- **Task Submissions**: What work is completed and submitted for review
- **Review Outcomes**: Approval vs. rejection decisions
- **Performance Metrics**: Completion rates, on-time delivery, quality
- **Activity Patterns**: General work patterns and consistency
- **Calendar & Meetings**: Meeting attendance and participation

### What We DON'T Monitor
- **Computer Activity**: We don't track keystrokes, mouse movement, or application usage
- **Screenshots/Screen Recording**: No automatic screenshots or screen recording
- **Websites Visited**: Not monitoring browsing history or websites visited
- **Personal Communications**: Email or messages not related to work are not monitored
- **Phone Calls**: Personal phone calls are private
- **Physical Location**: Where in your home or office you are working

### Monitoring Scope
- Monitoring is limited to **work activity only** during **working hours**
- Personal behavior and non-work activities are not monitored
- Focus is task completion and time accuracy, not surveillance

---

## Data Collected

### Work-Related Data
1. **Task Data**
   - Task assignment and status
   - Task submissions and timestamps
   - Submission review outcomes
   - Task completion rates
   - Task priorities and deadlines

2. **Time Tracking Data**
   - Timer start and stop times
   - Manual time entry start/end times
   - Total hours logged per day
   - Task association for time entries
   - Time entry edits and corrections

3. **Performance Data**
   - Task completion count
   - On-time vs. late submissions
   - Approval vs. rejection rates
   - Quality metrics
   - Performance ratings

4. **Activity Data**
   - Login and logout times
   - Meeting attendance
   - Communication timestamps
   - System usage patterns
   - General availability

### Data NOT Collected
- Keystrokes or mouse movements
- Website URLs or browsing history
- Application usage details
- Audio or video recordings
- Personal messages or communications
- Phone call details
- Camera or screen monitoring

---

## Purpose of Monitoring

### Legitimate Business Purposes
1. **Verify Actual Work Effort**
   - Confirm that assigned tasks are being worked on
   - Identify gaps or blocked work
   - Understand actual effort required

2. **Ensure Accurate Time Tracking**
   - Support fair performance evaluation
   - Ensure accuracy of time records
   - Detect time logging discrepancies

3. **Support Fair Performance Evaluation**
   - Objective metrics for rating
   - Transparent evaluation process
   - Merit-based decisions

4. **Identify Risks**
   - Project timeline risks
   - Resource availability issues
   - Skill gaps or training needed
   - Team member concerns

5. **Improve Operations**
   - Understand team dynamics
   - Identify process improvements
   - Support better resource planning
   - Coaching and development

### NOT for Punishment or Micromanagement
- Monitoring is not used for excessive control
- Focus is on results, not constant oversight
- Context matters (working hard, learning, etc.)
- Reasonable exceptions are understood
- Trust is foundational

## Role-Based Visibility

### What Each Role Can See

**Team Members Can See**:
- Own task assignments
- Own time entries and daily summaries
- Own task submissions and feedback
- Own performance metrics and ratings
- Own feedback from Team Lead

**Team Lead Can See**:
- Team member task assignments
- Team member time entries
- Team member task submissions
- Team member performance metrics
- Team-level aggregates
- Cannot access personal communications

**Project Manager Can See**:
- Project team member activity
- Task completion and progress
- Team performance metrics
- Time allocation by resource
- Cannot access non-project staff data

**Admin Can See**:
- All user data, all tasks, all metrics
- Complete audit trails
- Cross-project views
- System-wide analytics
- Exception: Not personal communications

**HR Manager Can See**:
- Personnel and compliance related data
- Performance records (for evaluations)
- Compliance metrics
- Limited project details

## Data Security

### Protection Measures
- All data stored securely
- Encrypted in transit (HTTPS)
- Encrypted at rest
- Access restricted by role
- Audit logs track data access
- Regular security reviews

### Data Retention
- Time tracking data: 3+ years
- Task data: 3+ years
- Performance data: 7 years
- Audit logs: 3+ years
- Deleted after retention period

### Privacy
- Personal data minimized
- Collected only when necessary
- Used only for stated purposes
- Not shared externally

- Record conversations or calls

- Personal communications

## Data Collected
The system collects work-related information, including:
- Assigned tasks and task status updates
- Work session start and stop timestamps
- Total hours logged per day
- Task submissions and timestamps
- Submission review outcomes
- Approved vs. rejected tasks
- Activity notes and task progress comments
- Optional future: activity indicators or screenshots

## Purpose of Monitoring
Monitoring is used to:
- Verify that assigned tasks are being worked on
- Ensure accurate time tracking and attendance
- Support fair performance evaluation and merit decisions
- Detect gaps, idle time, or unusual work patterns
- Maintain transparency and accountability
- Identify resource constraints and team needs
- Track project progress and delivery risk

## Policy Principles
- Monitoring is limited to work activity only
- Personal or non-work-related behavior is not captured
- Data is used for accountability and coaching, not invasive surveillance
- Admin and Manager review focuses on task completion, time integrity, and productivity
- All monitoring is conducted in compliance with applicable privacy laws

## Transparency Requirements
Users must understand that work activity is recorded and reviewed:
- Time logs are visible to the user, their Team Lead, Manager, and Admin
- Task activity is auditable and traceable
- Performance reports reflect monitored data
- Feedback is based on documented, measurable data
- Users can request data review or clarification

## Role-Based Visibility

### Intern / Employee
- Can view own time logs and task history
- Can see own performance reports
- Notified when submissions are under review
- Receives feedback on evaluation

### Team Lead
- Can view team member time logs
- Can see team member task submissions
- Can access team performance dashboards
- Can flag questionable entries for review

### Manager
- Can view assigned teams' time logs
- Can see all task submissions from teams
- Can access team performance reports
- Can identify patterns and trends

### Admin
- Can view all user time logs
- Can access complete audit trails
- Can override or investigate any data
- Can export compliance reports

## Data Retention
- Time logs and task data retained for 3 years (minimum)
- Performance evaluations retained for personnel records
- Audit logs retained for compliance and legal purposes
- Archived reports accessible on demand

## Policy Limitations
Monitoring does NOT include:
- Personal communications unrelated to work
- Websites visited outside of work context
- Personal files or systems
- Screen captures without explicit consent (if implemented)
- Audio or video recording without notification

## Compliance and Governance
- This policy is reviewed annually
- Updates made as needed to reflect legal changes
- All monitoring tools comply with data protection regulations
- Users are notified of any monitoring changes
