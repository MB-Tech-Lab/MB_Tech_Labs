# Work Monitoring & Tracking Policy

## Purpose
This policy describes how MB Tech Labs monitors work activity and time tracking to ensure accountability, transparency, and fair performance evaluation.

## Monitoring Scope
Users are monitored during official working hours and task execution to confirm genuine work effort and track productivity.

## Data Collected
The system collects work-related information, including:
- Assigned tasks and task status updates
- Work session start and stop timestamps
- Total hours logged per day
- Task submissions and timestamps
- Submission review outcomes
- Approved vs. rejected tasks
- Activity notes and task progress comments
- Optional future: activity indicators or screenshots

## Purpose of Monitoring
Monitoring is used to:
- Verify that assigned tasks are being worked on
- Ensure accurate time tracking and attendance
- Support fair performance evaluation and merit decisions
- Detect gaps, idle time, or unusual work patterns
- Maintain transparency and accountability
- Identify resource constraints and team needs
- Track project progress and delivery risk

## Policy Principles
- Monitoring is limited to work activity only
- Personal or non-work-related behavior is not captured
- Data is used for accountability and coaching, not invasive surveillance
- Admin and Manager review focuses on task completion, time integrity, and productivity
- All monitoring is conducted in compliance with applicable privacy laws

## Transparency Requirements
Users must understand that work activity is recorded and reviewed:
- Time logs are visible to the user, their Team Lead, Manager, and Admin
- Task activity is auditable and traceable
- Performance reports reflect monitored data
- Feedback is based on documented, measurable data
- Users can request data review or clarification

## Role-Based Visibility

### Intern / Employee
- Can view own time logs and task history
- Can see own performance reports
- Notified when submissions are under review
- Receives feedback on evaluation

### Team Lead
- Can view team member time logs
- Can see team member task submissions
- Can access team performance dashboards
- Can flag questionable entries for review

### Manager
- Can view assigned teams' time logs
- Can see all task submissions from teams
- Can access team performance reports
- Can identify patterns and trends

### Admin
- Can view all user time logs
- Can access complete audit trails
- Can override or investigate any data
- Can export compliance reports

## Data Retention
- Time logs and task data retained for 3 years (minimum)
- Performance evaluations retained for personnel records
- Audit logs retained for compliance and legal purposes
- Archived reports accessible on demand

## Policy Limitations
Monitoring does NOT include:
- Personal communications unrelated to work
- Websites visited outside of work context
- Personal files or systems
- Screen captures without explicit consent (if implemented)
- Audio or video recording without notification

## Compliance and Governance
- This policy is reviewed annually
- Updates made as needed to reflect legal changes
- All monitoring tools comply with data protection regulations
- Users are notified of any monitoring changes
